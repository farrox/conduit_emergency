Usage Guide

Basic Usage

# Start with default settings
conduit start --psiphon-config ./psiphon_config.json

# Customize limits
conduit start --psiphon-config ./psiphon_config.json --max-clients 500 --bandwidth 10

# Verbose output (info messages)
conduit start --psiphon-config ./psiphon_config.json -v

# Debug output (everything)
conduit start --psiphon-config ./psiphon_config.json -vv

Command Options

Flag Default Description
--psiphon-config, -c - Path to Psiphon network configuration file
--max-clients, -m 200 Maximum concurrent clients (1-1000)
--bandwidth, -b 5 Bandwidth limit per peer in Mbps (1-40)
--data-dir, -d ./data Directory for keys and state
-v - Verbose output (use -vv for debug)

Examples

High-Capacity Node

conduit start \
  --psiphon-config ./psiphon_config.json \
  --max-clients 1000 \
  --bandwidth 40

Low-Resource Node

conduit start \
  --psiphon-config ./psiphon_config.json \
  --max-clients 50 \
  --bandwidth 2

Custom Data Directory

conduit start \
  --psiphon-config ./psiphon_config.json \
  --data-dir /var/lib/conduit

Optimal Configuration

For the easiest way to configure optimal settings for maximum Iranians:

# Interactive configuration helper
./scripts/configure-optimal.sh

# Or quick start with auto-detection
./scripts/quick-optimal.sh

These scripts will automatically calculate the best max-clients and bandwidth settings based on your available bandwidth.

See CONFIG_OPTIMAL.md for detailed guidance and formulas.

Monitoring

Use verbose flags to monitor your node's activity:

  • -v - Shows info-level messages
  • -vv - Shows debug-level messages (very verbose)

Live Dashboard

One dashboard works for Docker and native Conduit. View it with:

./scripts/dashboard.sh

Start Conduit + dashboard (from source): ./scripts/start-with-dashboard.sh or ./scripts/test-option2-dashboard.sh (Option 2–style, one terminal).

Live Dashboard (Docker or native)

The dashboard shows: Online/Offline, CPU/RAM, connected users, Up/Down traffic, and refreshes every 5 seconds.
